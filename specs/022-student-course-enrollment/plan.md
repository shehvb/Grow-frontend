# Implementation Plan: Student Course List and Enrollment

**Branch**: `022-student-course-enrollment` | **Date**: 2026-05-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/022-student-course-enrollment/spec.md`

## Summary

This phase integrates the Student Dashboard's core learning modules with the backend APIs. We will connect the frontend to `/api/v1/courses/` to display enrolled courses, implement course discovery and enrollment via `/api/v1/courses/{id}/enroll/`, and enable lesson viewing through `/api/v1/courses/{id}/lessons/`. The implementation will heavily rely on Zustand for state management and `apiClient` for data fetching.

## Technical Context

**Language/Version**: TypeScript 5.x, React 18
**Primary Dependencies**: React Router DOM v6, Zustand, Axios (`apiClient`), TailwindCSS
**Storage**: Zustand memory store
**Testing**: Manual E2E validation against the local/remote PythonAnywhere backend
**Target Platform**: Web Browser (Desktop and Mobile responsive)
**Project Type**: Frontend Single Page Application (SPA)
**Performance Goals**: API calls must resolve within <2s, UI must not block during fetching
**Constraints**: Must handle JWT Bearer token authentication gracefully
**Scale/Scope**: 3 API endpoints, 3 main UI components (`StudentCourseList`, Enrollment Button, `CourseDetailsPage`).
